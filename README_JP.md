## テストポリシー

テストケースは[こちら](documents/tests/)

-   **Models**  
    モデルテストでは、モデルの属性やメソッドが正しく動作しているかを検証する。  
    API 経由での動作確認はシリアライザやビューのテストで行うため、ここでは対象外とする。

-   **Serializers**  
    シリアライザテストでは、モデルインスタンスとプリミティブなデータ型（例：辞書やリスト）との変換処理が正しく行われるかを検証する。また、デシリアライズ時のバリデーションロジックが期待通りに動作することも確認する。認証や権限の検証はビューの責務であるため、ここでは対象外とする。

-   **Views**  
    ビューテストでは、実際の HTTP リクエストとレスポンスを通じて API の挙動が正しいことを検証する。認証、権限、ルーティング、シリアライザの連携が期待通りに動作することを確認する。単体テストというよりも、統合テストに近い位置づけである。

## JWT 認証

-   クライアントはアクセストークンが有効な限り認証済みとなる
-   アクセストークンの有効期限が切れても、リフレッシュトークンが有効であれば新しいアクセストークンを取得できる
-   両方のトークンが期限切れになった場合、ユーザー名とパスワードで再度ログインし、新しい JWT を取得する必要がある

## トークンの有効期限

SimpleJWT のデフォルト設定はセキュリティ重視で、一般的な Web アプリケーションにはやや厳しい。本プロジェクトではセキュリティと利便性のバランスを考慮し、以下のように設定している。

-   **アクセストークン:** 15 分
-   **リフレッシュトークン:** 7 日間
